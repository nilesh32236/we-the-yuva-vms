## Review of Actual Changes (Backend Test Infrastructure)

### Strengths
- Adding a centralized test setup file is a reasonable direction for test environment configuration.
- The `.env.test` file provides clear documentation of expected test environment variables.

### Issues

#### Critical (Must Fix)
- **PR description mismatch**: The title and body describe frontend button component refactoring, but the diff contains only backend test files. This must be corrected before merge — either include the described frontend changes or update the PR title/description to match the actual diff.

#### Important (Should Fix)
- **`backend/setup-test.ts:1` — Unused import**: `beforeAll` is imported from `"vitest"` but never used. The file sets `process.env` at module scope, not inside a `beforeAll` callback. Biome's `noUnusedImports` rule will flag this as a warning.
- **`backend/setup-test.ts:2-5` — Duplicated configuration**: Values are hardcoded in `setup-test.ts` while identical values exist in `.env.test`. Nothing loads `.env.test` at runtime, making it dead documentation. If env values change, both files must be updated — a maintainability risk.
- **Conflicting test env pattern**: The existing test file (`backend/src/__tests__/e2e-flows.test.ts:4`) uses `vi.mock('@/config/env', () => ({...}))` to mock env vars at the module level. The `setup-test.ts` approach (setting `process.env` globally) introduces a second, conflicting pattern. Tests relying on the setup file may behave differently when run in isolation vs as part of a suite.

#### Minor (Nice to Have)
- **`backend/setup-test.ts:1-5` — Quote style**: Uses double quotes (`"vitest"`, `"postgresql://..."`) but the backend biome config specifies `"quoteStyle": "single"` (biome.json:54). Biome format will convert these on save.

### Assessment
**Ready to merge?** No

**Reasoning:** The PR description fundamentally misrepresents the actual diff — it claims to replace frontend buttons but only changes backend test files. The backend changes themselves have an unused import, duplicated config between files, and introduce a conflicting test env setup pattern.
