## Issues

### Minor (Nice to Have)

- **`events/page.tsx:32` — `active-bounce` removed**: The original Export CSV button had `active-bounce` on click; the Button's `outline` variant doesn't include it. This is consistent with the design system (only `primary` and `cta` variants bounce), but differs from the original behavior.
- **`coordinators/page.tsx:139-140` — Cancel button visual change**: The old Cancel button used muted styling (`text-brand-muted hover:text-brand-text`), while `variant="ghost"` makes it `text-brand-primary`. This makes the Cancel action more visually prominent — typically Cancel should be de-emphasized. Consider whether `variant="outline"` without the prominent text color would be more appropriate.
- **Buttons lose `font-semibold`**: The old "Add Coordinator" and "Save Coordinator" buttons used `font-semibold`; the Button component's base uses `font-medium`. This is a minor weight change across these two buttons, consistent with the design system.

## Recommendations

1. Check whether the Cancel button in the coordinators form should use `variant="outline"` instead of `variant="ghost"` to preserve the original muted/subtle visual hierarchy.
2. If `active-bounce` is desired on the Export CSV button, pass it in the `className` prop.
3. No critical or blocking issues found.

**Reasoning:** All 7 button replacements are correct, use appropriate variants, preserve existing functionality, and the `loading` prop is a genuine UX improvement.
