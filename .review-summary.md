## Improvements Made

### 1. Dark mode fix — `hover:bg-red-700` → `hover:opacity-90`

**File:** `frontend/components/ui/Button.tsx:14`

`hover:bg-red-700` is a raw hex color (`#b91c1c`). In dark mode, `--clr-brand-error` is `#f87171` (light red), so `hover:bg-red-700` would produce a very dark red hover state on an already-dark background, making text illegible. The project conventions (AGENTS.md) mandate theme tokens — "never raw color values."

Replaced with `hover:opacity-90`, which is used elsewhere in the codebase for destructive button hover states (e.g., `frontend/app/(coordinator)/coordinator/events/page.tsx:224`, `frontend/app/(coordinator)/coordinator/opportunities/page.tsx:200`). This works correctly in both light and dark modes.

### 2. Reject button loading state mismatch

**File:** `frontend/app/(admin)/admin/organizations/[id]/page.tsx:314`

The Reject button used `disabled={verifyMut.isPending}` while all nearby buttons (Approve, Suspend, Reactivate) use `loading={verifyMut.isPending}`. This created inconsistent UX — clicking Reject just greys out the button with no spinner feedback.

Changed to `loading={verifyMut.isPending}` for consistency.

### 3. Redundant className on stories buttons

**File:** `frontend/app/(admin)/admin/stories/[id]/page.tsx:173,187`

Both action buttons had `className="inline-flex items-center gap-2"` which is already applied by the `Button` component at `Button.tsx:51`. Removed.

### 4. Redundant conditional guard on icon rendering

**File:** `frontend/app/(admin)/admin/stories/[id]/page.tsx:175`

The `!moderateMut.isPending &&` guard around icon rendering is unnecessary — the `Button` component already hides `children` and shows a spinner when `loading` is true (`Button.tsx:59-63`). Removed the guard.

### 5. Redundant `variant="primary"` on users page

**Files:** `frontend/app/(admin)/admin/users/page.tsx:157,202`

`variant="primary"` is redundant since `primary` is the default value in the `Button` component (`Button.tsx:34`). Removed both instances.

## Unresolved Concerns

### Minor — ConfirmDialog variant naming mismatch

**File:** `frontend/components/admin/ConfirmDialog.tsx:11,59`

The `ConfirmDialog` accepts `variant?: 'danger' | 'primary'` but the `Button` component uses `'destructive'`. The mapping `variant === 'danger' ? 'destructive' : 'primary'` works correctly but is confusing for maintainers. Ideally `ConfirmDialog`'s prop should match `Button`'s naming (`'destructive'`), but this would require updating `frontend/app/(admin)/admin/stories/page.tsx:132` which is outside the PR scope.

### Minor — Unrelated change to backend/vitest.config.ts

**File:** `backend/vitest.config.ts:12-17`

Adds test environment variables (`DATABASE_URL`, `JWT_ACCESS_SECRET`, etc.) that are unrelated to the PR description ("Replace raw buttons with Button component"). This is a valid change but should have been in its own commit or mentioned in the PR description.
