## Review of Actual Changes

### Strengths
- Deleting `frontend/proxy.ts` (`frontend/proxy.ts:0`) is correct — it's dead code. No imports reference it, and `frontend/middleware.ts` handles the middleware logic. Clean removal of a stale file.
- Adding a centralized test setup file via `vitest.config.ts` is a reasonable direction for test infrastructure.

### Issues

#### Critical (Must Fix)
- **`PR:0` — PR description completely misrepresents the diff**: Title says "Replace raw buttons with custom Button component" across 3 org-admin pages, but those pages are identical to `origin/main`. The actual diff is purely backend test infrastructure + a dead file deletion. Merge without correction would leave the described work unapplied and confuse anyone reading git history. Either restore the button changes, or update the title/body to reflect the actual backend test infra work.

#### Important (Should Fix)
- **`backend/tests/setup.ts:1` — Unused `beforeAll` import**: `beforeAll` is imported from `"vitest"` but never called. The env assignments happen at module scope, not inside a `beforeAll` callback. This is visibly dead code that Biome's `noUnusedImports` rule would flag. Remove the import.
- **`backend/tests/setup.ts:2-5` — No `.env.test` loading at runtime**: `backend/.env.test` was added in `2e0c26a` then deleted in `3eef436`. Now the values live only as hardcoded strings in `setup.ts`. If values change, they must be edited in source. Consider using `dotenv` to load `.env.test` at test time instead, matching industry convention.
- **Conflicting test env pattern**: All existing tests (`backend/src/__tests__/e2e-flows.test.ts:4`) use `vi.mock('@/config/env', () => ({...}))` to mock the env config module. The new approach sets `process.env` globally at module scope. Tests relying on global `process.env` will behave differently when run in isolation vs as part of a suite, and global env mutations can leak across test files, causing nondeterministic failures.

#### Minor (Nice to Have)
- **`backend/tests/setup.ts:1-5` — Quote style mismatch**: Uses double quotes (`"vitest"`, `"postgresql://..."`) but `backend/biome.json:54` specifies `"quoteStyle": "single"`. Biome format will convert these on save.

### Recommendations
1. Align the PR title/description with the actual diff, or restore the button changes.
2. In `setup.ts`, remove the unused `beforeAll` import and switch to single quotes.
3. Consider loading env vars from a `.env.test` file using `dotenv` config, rather than hardcoding them in source.
4. Document whether existing `vi.mock('@/config/env')` tests are expected to coexist with this new `process.env` setup, or migrate them to the setup file for consistency.

### Assessment
**Ready to merge?** No
**Reasoning:** The PR description is fundamentally disconnected from the actual diff — it describes frontend button refactoring but the diff contains only backend test infrastructure changes plus a dead file deletion. Even ignoring the description gap, the new `setup.ts` file has an unused import, hardcoded values without env-file loading, and introduces a test configuration pattern that conflicts with the approach used in all 50+ existing tests.
